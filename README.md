# 작가 사이트 — CV / Works / Text / Contact

빌드 도구 없이 HTML/CSS/JS만으로 만든 사이트입니다. 해시(`#/works` 등) 기반
라우팅이라 GitHub Pages/Netlify 어디에 올려도 깨지지 않습니다.

## 폴더 구조
```
index.html          사이트 뼈대 (헤더, 라우팅 컨테이너, 라이트박스)
css/style.css        전체 스타일 (색상·폰트 토큰은 파일 상단 :root)
js/data.js           모든 실제 내용 — 여기만 수정하면 됩니다
js/app.js            라우터 + 화면 렌더링 + 검색 + 번역 로직 (보통 안 건드려도 됨)
images/               실제 이미지 파일을 넣는 폴더
```

## 화면 구조
- **홈 (`#/`)** = CV. 상단에 이름/생년월일/학력, 그 아래 개인전·그룹전·비평으로
  나뉜 전시 이력이 클릭 없이 한 화면에 다 보입니다. 전시명 클릭 → 상세 페이지.
- **전시 상세** = 서문(참조) → 전시 전경 → 작업 → 평문(참조). 내용이 없는
  섹션은 자동으로 화면에서 사라집니다.
- **Works (`#/works`)** = 카테고리 트리 → 최종 하위 카테고리에서 작품 썸네일 →
  클릭 시 큰 이미지(라이트박스).
- **Text (`#/text`)** = 카테고리 트리(내가 쓴 글/외부인 글 → 서문/평문/책) →
  글 목록 → 글 본문.
- **Search (`#/search`)** = Works/Text/Exhibition 통합 검색.
- **Contact (`#/contact`)** = 원하는 연락 수단을 자유롭게 추가 가능한 목록.
- 헤더 오른쪽 **EN/KO 버튼**으로 전체 사이트 언어 전환.

## 1. 로컬에서 확인하기
`index.html` 더블클릭하면 바로 열립니다.

## 2. 한/영 전환 (번역)
자동번역이 아니라 **회찬 님이 두 언어를 직접 등록**하는 방식입니다.
`js/data.js`에서 텍스트가 들어가는 자리는 대부분 이렇게 생겼습니다:
```js
title: { ko: "무제", en: "Untitled" }
```
- `en`을 비워두면(`""`) 그 항목은 영어 모드로 바꿔도 한글 그대로 남습니다
  (번역 안 한 항목만 안 바뀌고, 나머지는 정상적으로 바뀝니다).
- `year`, `size`, 이미지 경로처럼 언어가 상관없는 값은 이중언어
  객체가 아니라 그냥 문자열입니다. 실수로 `{ ko, en }`을 안 씌워도 됩니다.
- Works, Text, Contact, "연작 위치 찾아보기" 같은 사이트 고정 문구는 이미
  다 번역이 들어가 있어서 신경 안 써도 됩니다 (`js/data.js` 맨 아래 `UI`
  부분인데, 여기는 안 건드려도 됩니다).

## 3. Works — 카테고리 트리 (`WORK_CATEGORIES`)
한 노드는 둘 중 하나만 가집니다.
- `children` 배열 → 하위 카테고리 목록으로 표시
- `works` 배열 → 작품 썸네일 그리드로 표시 (`WORKS`의 id를 나열)

더 깊게 나누고 싶으면 `works` 대신 `children`을 넣고 그 안에 같은 형태의
노드를 추가하면 됩니다. 깊이 제한 없습니다. 한 카테고리에 작품이 24개를
넘으면 자동으로 페이지가 나뉩니다.

## 4. Text — 카테고리 트리 (`TEXT_CATEGORIES`) + 글 원본 (`POSTS`)
Works와 완전히 같은 규칙입니다. 글 본문 안에서 `[보일 텍스트](URL)` 형식으로
쓰면 자동으로 하이퍼링크가 됩니다 (주황색, 새 탭으로 열림). "책" 카테고리에
구매 링크를 걸 때 이 문법을 쓰면 됩니다. 한 카테고리에 글이 20개를 넘으면
자동으로 페이지가 나뉩니다.

## 5. CV — 프로필 / 전시 분류
```js
const PROFILE = {
  name: { ko: "권회찬", en: "Kwon Hoechan" },
  birth: { ko: "1995년 3월 12일생", en: "Born March 12, 1995" },
  education: [
    { ko: "OO대학교 미술대학 회화과 졸업", en: "B.F.A. in Painting, OO University" }
  ]
};
```
전시 항목마다 `type` 필드로 분류합니다: `"solo"`(개인전) / `"group"`(그룹전)
/ `"critic"`(비평). CV 홈 화면에 이 순서대로 소제목이 나뉘어 다 보입니다.

## 6. 전시(CV)에 서문·평문 연결하기
```js
{
  id: "ex-2023-doodle",
  type: "solo",
  title: { ko: "낙서의 초상", en: "Portraits of Doodle" },
  year: "2023",
  venue: { ko: "에브리아트", en: "Every Art" },
  essayPostId: "post-01",        // 서문 — 하나만 (없으면 "")
  criticPostIds: ["post-02"],    // 평문 — 여러 개 가능 (없으면 [])
  installationImages: [...],
  works: ["p01", "p02"]
}
```
`essayPostId`, `criticPostIds`, `installationImages`, `works` 중 비어있는
항목은 전시 상세 페이지에서 그 섹션 자체가 자동으로 안 보입니다.

## 7. 라이트박스 (작품 큰 이미지)
- **마우스 휠**: 위로 굴리면 확대, 아래로 굴리면 축소 (최대 4배)
- **드래그**: 확대된 상태에서 이미지를 눌러서 끌면 이동
- **연작/전시 링크**가 문맥에 따라 자동으로 바뀝니다:
  - 전시 상세 페이지의 "작업"에서 열면 → **"연작 위치 찾아보기"** (Works로 이동)
  - Works 카테고리 썸네일에서 열면 → **"전시 찾아보기 1, 2..."** (그 작품이
    걸렸던 전시들로 이동, 최신 연도 순). 전시에 한 번도 안 걸린 작품이면
    링크 자체가 안 뜹니다.

## 8. 검색 (`#/search`)
Works, Text, Exhibition 세 카테고리로 나눠서 결과를 보여줍니다.
- 검색어와 일치하는 부분은 주황색으로 강조됩니다.
- **카테고리 이름 자체(예: "낙서의 초상", "초상")만 걸리면 결과에서 제외**되고,
  그 안의 실제 작품/글까지 일치해야 결과에 나옵니다.
- Works 결과: 작품 제목은 항상 표시, 그 아래 매체/연도/사이즈/note 중
  일치한 것 하나를 표시 (여러 개 겹치면 연도→매체→사이즈→note 순으로 하나만).
- Text 결과: 글 제목은 항상 표시, 그 아래 필자/날짜/본문 중 일치한 것
  하나를 표시. 본문이 길면 일치한 부분 앞뒤로 문맥을 잘라서 보여줍니다.
- Exhibition 결과: 연도나 장소가 걸리면 연도·장소·전시명을 함께 표시.
- 검색 결과가 50개를 넘으면 페이지가 나뉩니다.

## 9. 이미지 두 단계로 넣기 (저용량 썸네일 / 고용량 원본)
```js
c01: { thumb: "images/composition-01-thumb.jpg", image: "images/composition-01.jpg", ... }
```
- **`thumb`** — 그리드 표시용 저용량. 긴 변 600~800px, 100~200KB 이하 목표.
- **`image`** — 클릭 시 보이는 고용량 원본. 긴 변 1600~2000px 정도.
- `thumb`을 비워두면 자동으로 `image`를 대신 씁니다. 둘 다 비워두면
  자리표시자(해칭 패턴)가 표시됩니다.
- 썸네일 만들기: https://squoosh.app (드래그만 하면 리사이즈+압축 자동)
- 파일명은 영문+하이픈으로 짧게 쓰는 걸 권장합니다 (`self-portrait-01.jpg`).
- 그리드에 뜨는 비율은 **이미지 원본 비율을 자동으로 사용**합니다. 별도로
  비율 값을 입력할 필요 없이, 파일을 그대로 넣으면 잘리지 않고 원본
  생긴 대로 나옵니다. 이미지가 아직 없는 자리(자리표시자)만 임시로
  고정된 비율(4:5)을 씁니다.

## 10. Contact 항목 추가/삭제
`CONTACT`는 목록입니다. 줄을 추가하거나 지우면 Contact 페이지에 그대로
반영됩니다.
```js
const CONTACT = [
  { label: "Email", value: "contact@example.com", href: "mailto:contact@example.com" },
  { label: "Instagram", value: "Instagram", href: "https://instagram.com/example" },
  { label: "Threads", value: "Threads", href: "https://threads.net/@example" }
];
```
- `href`가 `mailto:`나 `tel:`로 시작하면 그 자리에서 바로 연결(메일 앱, 전화
  앱 실행), 그 외(`https://` 등)는 새 탭에서 열립니다.
- `value`가 `label`과 다르면 "Email: contact@example.com"처럼 같이
  표시되고, 같으면(Instagram처럼) 라벨만 표시됩니다.

## 11. 색상/폰트 바꾸기
`css/style.css` 맨 위 `:root { ... }` 값만 바꾸면 전체 톤이 바뀝니다.

## 12. 무료로 배포하기

### GitHub Pages
1. GitHub에 새 저장소를 만들고 이 폴더 전체를 업로드합니다.
2. Settings → Pages → Branch를 `main`으로 선택 후 저장.
3. `https://[아이디].github.io/[저장소이름]` 로 접속됩니다.

### Netlify (더 간단)
1. https://app.netlify.com/drop 접속
2. 폴더를 통째로 드래그 앤 드롭 → 바로 배포됩니다.

### 커스텀 도메인
가비아/후이즈/Namecheap 등에서 도메인 구매(연 1~2만원대) 후, 호스팅
서비스의 "Custom domain" 설정에서 안내하는 DNS 레코드를 등록업체
관리 페이지에 추가하면 연결됩니다.

## 13. 업데이트하는 법
로컬에서 `data.js` 등을 수정하는 것만으로는 실제 사이트(인터넷 주소)에
반영되지 않습니다. 수정 후 다시 호스팅 서비스에 업로드(재배포)해야
반영됩니다.
- Netlify 드래그앤드롭 방식이면 매번 폴더를 다시 드래그
- GitHub Pages면 수정한 파일을 저장소에 다시 올려야 함
- GitHub + Netlify를 연결해두면, GitHub에 올리는 순간 자동으로 재배포됨
